Command Dialog Layout

How should the Constraint Manager dialog be organized?

A

Compact — Single Panel

Constraint Manager
SELECT ENTITY
← Click a sketch entity in the viewport
CONSTRAINTS (5 found)
☐ Type Related To
☑ Coincident SketchPoint3
☐ Horizontal —
☐ Parallel Line3
☐ Equal Line5
🔒 Tangent Arc2 (not deletable)

Everything in one view. Selection input on top, constraint table below. Minimal, functional.

B

Grouped — Geometric vs Dimensional

Constraint Manager
SELECT ENTITY
Line1 (SketchLine)
▼ GEOMETRIC CONSTRAINTS (3)
☐ Coincident SketchPoint3
☐ Horizontal —
☐ Parallel Line3
▼ DIMENSIONS (2)
☐ Distance Line3 — 25.4mm
☐ Angular Line7 — 45.0°

Same idea but splits geometric constraints from dimensional ones. Clearer when there are many constraints, but adds visual complexity.

Recommendation: A

  • Fusion's TableCommandInput doesn't natively support collapsible groups — B would require workarounds
  • Constraint type column already tells you if it's geometric vs dimensional
  • Simpler to build, easier to use

When B would matter

  • Entities with 10+ constraints might benefit from grouping
  • Could add later as a filter dropdown instead of grouped sections
